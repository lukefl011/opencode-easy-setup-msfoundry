# Context Profile: Web

Use this profile for browser and frontend web application work.

## Objectives

- deliver accessible, responsive, and reliable user experiences
- preserve performance budgets across key user journeys
- keep component and data boundaries explicit and maintainable

## Planning expectations

- include page/feature impact map and routing implications
- include API contract impact and fallback behavior
- include accessibility, performance, and security considerations
- include unit/component/e2e smoke strategy

## Build/check expectations

- lint, typecheck, unit tests, component tests
- browser smoke checks for critical paths

## Release expectations

- environment and runtime config completeness
- cache, CDN, and asset versioning readiness
- monitoring, analytics, and error tracking verification
- rollback plan for frontend and API compatibility
